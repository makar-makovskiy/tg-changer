
Ниже — практичная инструкция по запуску **на сервере через Docker Compose** (как задумано в проекте).

---

## Что должно быть на сервере

- **Linux** (часто Ubuntu/Debian) с правами `sudo` или root  
- Установлены **Docker** и плагин **Compose** (`docker compose version` показывает версию)  
- Открыты порты в **файрволе** (см. ниже)  
- Код проекта на машине (через `git clone` или копирование архива)

---

## 1. Получить проект на сервер

```bash
cd /opt   # или другая папка
git clone <url-репозитория> servjs
cd servjs
```

Если репозитория нет — скопируй папку проекта целиком, чтобы сохранились `serv/server`, `serv/client`, `docker-compose.yml`, `.postgres.env`.

---

## 2. Пароли базы (обязательно для продакшена)

Файл **`.postgres.env`** задаёт пользователя и пароль Postgres. Сейчас там учётка **`app` / `app` / база `app`** — для публичного сервера **смени пароль** (и при желании имя пользователя/БД).

В **`docker-compose.yml`** у сервиса `backend` переменная **`DATABASE_URL`** должна **совпадать** с тем, что в `.postgres.env`:

```text
postgresql://ПОЛЬЗОВАТЕЛЬ:ПАРОЛЬ@db:5432/ИМЯ_БД
```

Хост **`db`** — это hostname контейнера Postgres в Compose, его не меняй.

---

## 3. Запуск

Из **корня проекта** (где лежит `docker-compose.yml`):

```bash
docker compose up -d --build
```

- **`-d`** — в фоне. Без `-d` логи идут в консоль (удобно для отладки первого запуска).

Проверка:

```bash
docker compose ps
```

---

## 4. Что открыть в браузере

| Сервис | URL | Назначение |
|--------|-----|------------|
| Приложение (UI) | `http://IP-СЕРВЕРА:3000` | Страница со списком пользователей |
| API (опционально) | `http://IP-СЕРВЕРА:5000` | Прямой доступ к Express |
| Adminer | `http://IP-СЕРВЕРА:8080` | Веб-админка БД |
| Postgres | порт **5432** | Подключение клиентами БД |

Для обычных пользователей достаточно **порта 3000**.

---

## 5. Файрвол

Пример для **ufw** (Ubuntu):

```bash
sudo ufw allow 22/tcp      # SSH
sudo ufw allow 3000/tcp    # сайт
sudo ufw enable
```

**Adminer (8080)** и **Postgres (5432)** лучше **не выставлять в интернет**: либо не открывать порты, либо доступ только с VPN/по SSH-туннелю. Если Adminer не нужен на проде — можно убрать сервис `adminer` из `docker-compose.yml` или не публиковать его порт.

---

## 6. Обновление после изменений в коде

```bash
cd /path/to/servjs
git pull   # если используешь git
docker compose up -d --build
```

---

## 7. HTTPS и домен (по желанию)

Сейчас трафик идёт по **HTTP**. Для **HTTPS** обычно ставят на сам сервер **Caddy** или **Nginx** как reverse proxy: слушает **443**, выдаёт сертификат (Let’s Encrypt), проксирует на `127.0.0.1:3000`. Тогда наружу открывают **80** и **443**, порт **3000** можно оставить только для localhost.

---

## 8. Если Docker на сервере не используешь

Нужны **Node.js**, **PostgreSQL** на хосте, в **`serv/server/.env`** — свой `DATABASE_URL`, установка зависимостей (`npm install`), миграции (`npx prisma migrate deploy`), запуск `node server.js` (или `npm start` из корня репозитория). Для продакшена обычно ставят **systemd** или **pm2**, чтобы процесс перезапускался. Это отдельный сценарий от Docker-инструкции выше.

---

Кратко: на сервере клонируешь проект, **согласуешь пароли** в `.postgres.env` и `DATABASE_URL`, выполняешь **`docker compose up -d --build`**, открываешь **порт 3000** — сайт доступен по `http://IP:3000`. Если нужно, могу отдельно расписать только вариант **без Docker** или только **HTTPS + домен** под твой Linux.
